# AI連携基盤（AIIntegration）

## 概要

各種AI機能を実現するための基盤設計・方針を定義するドキュメント。

## 設計方針

### 1. プライバシーファースト

- ローカル処理を優先
- 外部API利用時は明確な同意
- データの匿名化・最小化

### 2. オプトイン方式

- AI機能はデフォルトOFF
- ユーザーが明示的に有効化
- いつでも無効化可能

### 3. 透明性

- AIが生成した内容の明示
- 使用するモデル・サービスの開示
- コスト（あれば）の可視化

## 技術構成（案）

### 外部API方式

```
ノート → API呼び出し → LLM → 結果
```

- メリット：高性能、常に最新
- デメリット：コスト、プライバシー懸念

### ローカルLLM方式

```
ノート → ローカルLLM → 結果
```

- メリット：プライバシー、オフライン
- デメリット：ハードウェア要件、性能限界

### ハイブリッド方式

- 軽量処理：ローカル
- 高度な処理：外部API（同意の上）
