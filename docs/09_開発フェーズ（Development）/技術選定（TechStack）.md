# 技術選定

## 基本方針

- **シンプルさ優先**: 個人開発で管理可能な技術スタック
- **最新技術採用**: 積極的に新しい技術を試す
- **ローカルファースト**: データは手元に、クラウド依存しない

## 技術選定の判断基準

### 採用する場合

1. 開発体験が大幅に向上する
2. パフォーマンスが明確に改善する
3. 学習コストに見合う価値がある
4. コミュニティが活発で将来性がある

### 採用しない場合

1. 現状で十分機能している
2. 複雑性が増す割にメリットが少ない
3. 実験的すぎて安定性に不安
4. ロックインのリスクが高い
