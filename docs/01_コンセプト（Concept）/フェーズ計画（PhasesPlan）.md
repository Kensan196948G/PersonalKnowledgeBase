# フェーズ計画（PhasesPlan）

## 概要

システム開発を段階的に進めるためのフェーズ分けと、各フェーズで実現する機能を整理する。

## フェーズ構成

### フェーズ1：書くことに集中

**目標**: 基本的なメモ作成・保存ができる状態

| 機能 | 優先度 |
|------|--------|
| 折りたたみブロック | 高 |
| 引用ブロック | 高 |
| テンプレート機能 | 中 |
| フォルダ管理 | 中 |

### フェーズ2：探しやすくする

**目標**: 蓄積したメモを効率的に検索・発見できる状態

| 機能 | 優先度 |
|------|--------|
| 高度検索 | 高 |
| タグ可視化 | 中 |
| お気に入り | 中 |
| 最近更新リスト | 低 |

### フェーズ3：知識として育てる

**目標**: メモ同士の関連性を活用し、知識を発展させられる状態

| 機能 | 優先度 |
|------|--------|
| ノート間リンク | 高 |
| バックリンク | 高 |
| 関連ノート提示 | 中 |
| 関係可視化 | 低 |

## フェーズ移行の判断基準

- 現フェーズの主要機能が安定動作している
- 日常的に使用してフィードバックを得ている
- 次フェーズの機能が必要と感じている
