# 基本思想（DesignPolicy）

## 概要

本システムの設計における基本的な考え方・ポリシーを定義する。

## 設計原則

### 1. 小さく始め、後から積み上げられる

- 最小限の機能からスタート
- 必要に応じて機能を追加
- 過度な先読み設計を避ける

### 2. 思考を邪魔しない UI / UX

- 書くことに集中できるインターフェース
- 余計な操作や確認を減らす
- 直感的な操作性

### 3. 書いたあとに"探せる"こと

- 検索性を重視した設計
- タグ・カテゴリによる整理
- 時系列での追跡

### 4. データは常に自分の手元にある

- ローカルファースト
- クラウド依存しない構造
- エクスポート・バックアップが容易

### 5. 将来の AI 連携を阻害しない構造

- 機械可読なデータ形式
- API 化を見据えた設計
- テキストベースの保存

## 避けるべきこと

- 過度な抽象化
- 使わない機能の先行実装
- 複雑な依存関係
